\section{Chi tiết triển khai}

\subsection{Giai đoạn 1: Bảo mật và Cấu hình Môi trường}

Bước đầu tiên là chuẩn bị môi trường Google Cloud để pipeline có thể chạy một cách an toàn. Chúng em tạo một dự án riêng biệt, kích hoạt các API cần thiết, và tạo một tài khoản dịch vụ có tên \texttt{cloud-function-sa} cho hàm được triển khai.

Các API sau được bật cho dự án:

\begin{itemize}
    \item Cloud Pub/Sub API
    \item Cloud Functions API
    \item BigQuery API
    \item Cloud Natural Language API
\end{itemize}

Các vai trò chính được gán cho tài khoản dịch vụ \texttt{cloud-function-sa} là:

\begin{itemize}
    \item \textbf{Pub/Sub Subscriber} --- cho phép hàm nhận sự kiện từ trigger Pub/Sub
    \item \textbf{BigQuery Data Editor} --- cho phép hàm chèn các bản ghi được làm giàu dữ liệu vào \texttt{analytics\_ds.website\_logs}
    \item \textbf{Cloud Functions Invoker} --- cho phép hàm chạy trong môi trường triển khai
    \item \textbf{Cloud Natural Language API User} --- cho phép truy cập API phân tích cảm xúc nếu cần thiết trong quá trình triển khai hoặc chạy
\end{itemize}

\subsubsection*{Tạo log simulator}

Chúng em tạo file \texttt{log\_simulator.py} trong Cloud Shell tạo ra các sự kiện website tổng hợp với các trường: \texttt{timestamp}, \texttt{user\_id}, \texttt{action}, \texttt{page}, \texttt{response\_time\_ms}, và tùy chọn \texttt{feedback} (khả năng 30\%). Script sử dụng Google Cloud Pub/Sub client library, được cài đặt qua:

\begin{lstlisting}[language=bash, frame=none]
pip install google-cloud-pubsub
\end{lstlisting}

\subsubsection*{Chạy simulator}

Chúng em đã chạy simulator trong Cloud Shell:

\begin{lstlisting}[language=bash, frame=none]
python log_simulator.py
\end{lstlisting}

Simulator đã xuất bản tin nhắn liên tục. Chúng em đã xác minh dữ liệu đang chảy bằng cách kéo tin nhắn từ subscription \texttt{website-logs-sub} trong Pub/Sub console và xác nhận tất cả các trường đều có mặt và được định dạng chính xác.

\subsection{Giai đoạn 3: Xử lý với Cloud Functions và AI}

Ở giai đoạn này, chúng em triển khai tầng xử lý dưới dạng dịch vụ Cloud Function có tên \texttt{process-website-logs} và kết nối với Pub/Sub để mỗi tin nhắn mới từ \texttt{website-logs} sẽ tự động kích hoạt xử lý.

Quy trình thiết lập được thực hiện theo các bước sau:

\begin{enumerate}
    \item Mở Cloud Run functions từ khu vực Cloud Functions trong Google Cloud Console.
    \item Tạo một Python function bằng inline editor.
    \item Thiết lập cấu hình dịch vụ cốt lõi:
    \begin{itemize}
        \item Service name: \texttt{process-website-logs}
        \item Runtime: Python (phiên bản mới nhất có trong console)
        \item Billing mode: Request-based
        \item Minimum instances: 0
    \end{itemize}
    \item Cấu hình trigger trong Eventarc Trigger:
    \begin{itemize}
        \item Event provider: Cloud Pub/Sub
        \item Event type: \texttt{google.cloud.pubsub.topic.v1.messagePublished}
        \item Topic: \texttt{website-logs}
        \item Region: cùng region với function khi có thể
    \end{itemize}
    \item Cập nhật thiết lập bảo mật runtime:
    \begin{itemize}
        \item Runtime service account: \texttt{cloud-function-sa}
        \item Function entry point: \texttt{process\_log}
    \end{itemize}
\end{enumerate}

Sau khi hoàn tất phần cấu hình, chúng em cập nhật mã nguồn trong inline editor:

\begin{itemize}
    \item \texttt{main.py}: nhận dữ liệu Pub/Sub, làm giàu sentiment bằng Natural Language API, và ghi dữ liệu đã làm giàu vào BigQuery.
    \item \texttt{requirements.txt}: khai báo các thư viện phụ thuộc cần khi deploy.
\end{itemize}

\begin{lstlisting}[frame=none]
functions-framework==3.*
google-cloud-bigquery==3.21.0
google-cloud-language==2.13.0
\end{lstlisting}

Tiếp theo, chúng em deploy function và kiểm tra deployment trong logs. Một lần chạy thành công cho thấy các bản ghi được xử lý và chèn vào \texttt{analytics\_ds.website\_logs}, xác nhận Giai đoạn 3 đã hoàn tất và sẵn sàng cho các bước phân tích downstream.

\subsection{Giai đoạn 4: Phân tích với BigQuery và BigQuery ML}

Chúng em tạo BigQuery dataset \texttt{analytics\_ds} và bảng \texttt{website\_logs} để lưu dữ liệu đã làm giàu từ Cloud Function. Schema của bảng được khai báo như sau:

\begin{lstlisting}[frame=none]
[
    { "name": "timestamp", "type": "TIMESTAMP", "mode": "NULLABLE" },
    { "name": "user_id", "type": "STRING", "mode": "NULLABLE" },
    { "name": "action", "type": "STRING", "mode": "NULLABLE" },
    { "name": "page", "type": "STRING", "mode": "NULLABLE" },
    { "name": "response_time_ms", "type": "INTEGER", "mode": "NULLABLE" },
    { "name": "feedback", "type": "STRING", "mode": "NULLABLE" },
    { "name": "sentiment_score", "type": "FLOAT", "mode": "NULLABLE" },
    { "name": "processed_at", "type": "TIMESTAMP", "mode": "NULLABLE" }
]
\end{lstlisting}

Sau khi bảng có đủ dữ liệu, chúng em huấn luyện mô hình logistic regression trong BigQuery ML với tên \texttt{purchase\_prediction}. Câu lệnh SQL dùng để huấn luyện mô hình như sau:

\begin{lstlisting}[language=SQL, frame=none]
CREATE OR REPLACE MODEL `analytics_ds.purchase_prediction`
OPTIONS(
    model_type='logistic_reg',
    input_label_cols=['is_purchase'],
    auto_class_weights=TRUE
) AS
SELECT
    IF(action='purchase',1,0) AS is_purchase,
    sentiment_score,
    response_time_ms
FROM `analytics_ds.website_logs`
WHERE sentiment_score IS NOT NULL;
\end{lstlisting}

Truy vấn trên sử dụng \texttt{action} để tạo nhãn \texttt{is\_purchase}, sau đó huấn luyện mô hình với hai đặc trưng đầu vào là \texttt{sentiment\_score} và \texttt{response\_time\_ms}.

\subsection{Giai đoạn 5: Trực quan hóa dữ liệu với Looker Studio}

Giai đoạn cuối cùng của pipeline là trực quan hóa kết quả phân tích. Chúng em kết nối Looker Studio trực tiếp với bảng \texttt{analytics\_ds.website\_logs} để xây dựng một dashboard thương mại tương tác.

\subsubsection*{Kết nối Looker Studio và BigQuery}

\begin{enumerate}
    \item Truy cập Looker Studio (looker.studio)
    \item Tạo một báo cáo mới (New Report)
    \item Chọn BigQuery làm nguồn dữ liệu
    \item Chọn dự án GCP, dataset \texttt{analytics\_ds} và bảng \texttt{website\_logs}
    \item Xác nhận kết nối
\end{enumerate}

\subsubsection*{Xây dựng các biểu đồ chính}

Chúng em xây dựng ba biểu đồ chính để trình bày thông tin kinh doanh quan trọng:

\begin{enumerate}
    \item \textbf{Biểu đồ Gauge (Sentiment Trung Bình)}
    \begin{itemize}
        \item Loại: Gauge (thước đo)
        \item Số liệu: Giá trị trung bình của \texttt{sentiment\_score}
        \item Mục đích: Hiển thị độ hài lòng chung của khách hàng
        \item Các bước thực hiện:
        \begin{itemize}
            \item Thêm một biểu đồ Gauge vào dashboard
            \item Chọn trường \texttt{sentiment\_score} làm số liệu
            \item Cấu hình phạm vi hiển thị từ 0 đến 1
            \item Chọn phép tính AVG cho \texttt{sentiment\_score} 
        \end{itemize}
        
    \end{itemize}
    \item \textbf{Biểu đồ Chuỗi Thời Gian (Lưu lượng theo Giờ)}
    \begin{itemize}
        \item Loại: Time series (đường biểu đồ)
        \item Trục X: Thời gian (timestamp)
        \item Trục Y: Số lượng sự kiện
        \item Mục đích: Cho thấy xu hướng lưu lượng truy cập theo thời gian
        \item Các bước thực hiện:
        \begin{itemize}
            \item Thêm một biểu đồ  chuỗi thời gian vào dashboard
            \item Chọn trường \texttt{timestamp} làm trục X và đếm số lượng sự kiện làm trục Y
            \item Cấu hình để trục X nhóm theo giờ
        \end{itemize}
    \end{itemize}
    
    \item \textbf{Biểu đồ Tròn (Phân bố Hành động)}
    \begin{itemize}
        \item Loại: Pie chart (biểu đồ tròn)
        \item Dữ liệu: Số lượng từng loại hành động (\texttt{action})
        \item Mục đích: So sánh tỷ lệ các hành động người dùng
        \item Các bước thực hiện:
        \begin{itemize}
            \item Thêm một biểu đồ tròn vào dashboard
            \item Chọn trường \texttt{action} làm phân loại và đếm số lượng sự kiện làm giá trị
            \item Cấu hình để hiển thị tỷ lệ phần trăm cho mỗi loại hành động
        \end{itemize}
    \end{itemize}
\end{enumerate}
